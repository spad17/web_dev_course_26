# Идеи для разработки гемов

## Доработать fast_matrix

https://github.com/mmcs-ruby/fast_matrix

Этот гем сделали студенты мехмата несколько лет назад. Гем использует нативные расширения на C для оптимизации операций над матрицами.
С тех пор вышло несколько новых версий Ruby и немного изменилось API для вызова нативных расширений. Было бы здорово обновить гем, чтобы он работал с последними версиями Ruby и поддерживал новые возможности языка.


В качестве эксперимента П.А. уже обновил гем для Ruby 4.0, можно консультироваться с ним по вопросам реализации.
Дуюнов С.И. также готов дать консультацию по этому проекту и провести ревью кода.

Сложность: средняя. Требуется знание C и Ruby, а также понимание работы с нативными расширениями.

## Разработать гем для работы с базой данных публикаций 

Есть сервис openalex.org, который предоставляет API для доступа к данным о научных публикациях. Было бы полезно создать гем, который бы оборачивал этот API и предоставлял удобный интерфейс для получения данных о публикациях, авторов, журналов и т.д.

Дополнительно хорошо бы иметь какие-то базовые функции для анализа данных, например, подсчет количества публикаций по годам, топ авторов и т.д.

## Гем для работы с pdf (для Linux)

Локально работающий аналог ilovepdf.com, который позволяет пользователю объединять, разделять, сжимать и конвертировать PDF файлы, а также перегонять изображения в pdf. 
Гем должен предоставлять простой интерфейс для выполнения этих операций.
Можно реализовать как обертку над существующими библиотеками для работы с PDF, например, pdftk или poppler.
В заголовке написано Linux, потому что на Windows собрать это будет сложнее, но не прям невозможно.

## Гем для работы с музыкой и/или звуком

Есть старый и заброшенный гем https://github.com/sausheong/muse
Он позволяет генерировать wav файлы на основе собственного DSL.
Нужно сделать аналог, который предложид бы новый DSL и работал бы быстрее.
Опционально можно добавить поддержку других форматов, например, mp3 или ogg, а также возможность работать с MIDI файлами.

В качестве референса можно рассмотреть еще и https://github.com/pedrozath/coltrane

Сложность: высокая. Требуется базовое знание теории музыки, а также опыт работы с аудио файлами и генерацией звука.

## Гем для работы с графами

Наименее востребованный гем, и уже несколько раз был сделан, но можно рассматривать его как способо прокачать свои знания в области алгоритмов и структур данных. Гем должен предоставлять удобный интерфейс для создания и работы с графами, а также реализовывать базовые алгоритмы, такие как поиск в глубину, поиск в ширину, алгоритм Дейкстры и т.д.
